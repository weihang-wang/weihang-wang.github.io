Course Description:

Today, more than 4 billion people worldwide are using the Internet in their everyday lives. Unfortunately, severe security flaws are commonly found in many web applications. With great regularity, we hear about web security breaches compromising millions of users’ personal data, or installing malicious software on users’ computers with no user interaction required. Over the past two decades, the research community has proposed various mechanisms to mitigate certain web vulnerabilities. With the evolution of web technologies, the arms race between researchers and adversaries continues.

In this seminar, we will discuss recent web security papers from top conferences. Topics include access control policies, web injection, vulnerable HTML5 features, privacy risks associated with third-party scripts, business logic flaws, session management flaws, side channel leaks, fingerprinting, web vulnerabilities testing, etc. The goal of this seminar is to learn the most recent security flaws and defense techniques in web applications, reason about possible shortcomings, and get hands-on experience in building effective defense mechanisms and understanding the latest cyberattacks.

Prerequisites:

  • Students need to have prior exposure to concepts including vulnerabilities, defenses, side-channel attacks, black box testing etc.
  • Basic understanding on simple program analysis (static analysis such as control flow and data flow analysis, dynamic analysis such as program instrumentation/transformation and profiling) and software engineering techniques are needed to help you understand the papers.
  • Basic knowledge or experience in web programming in JavaScript are required.

Course Structure:

This class has no exam. The grading is based on paper presentations and projects. Only satisfactory (S) and unsatisfactory (U) will be given.
Paper presentations: we will discuss 4 papers each week. Each student will be expected to present 1-2 papers and lead discussions. These papers are selected from recent publications from top conferences in security.
Projects: students who registered more than 1 credit need to complete a project. For the project, students are expected to demo the code implementation during office hours and write a report with a description of how to run the code. A template for the report will be given. Students are also welcome to propose their own project. Please come to the instructor’s office hours or make an appointment to discuss the topic of the project.
